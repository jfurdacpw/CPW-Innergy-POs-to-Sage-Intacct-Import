Blocked on a Sage ID

These columns stay blank until the matching Sage identifier is available in Innergy — either set on a record’s External Id, or mapped somewhere the export can read. None of these are code problems: each is waiting on an ID or a decision, then it’s a small change.

FieldExportSage ID neededHow it gets unlocked
CUSTOMER_IDAR Customer ID Put the Sage customer ID in each customer’s External Id field in Innergy — the export links it by name automatically.
ARINVOICEITEM_CUSTOMERIDAR Customer ID Same customer link as CUSTOMER_ID, applied to the line’s customer dimension.
VENDOR_IDAP Vendor ID Put the Sage vendor ID in each vendor’s External Id field in Innergy.
DEPT_IDAR + AP Department ID Map Innergy → Sage departments (e.g. Furniture, Millwork) somewhere the export can read.
LOCATION_IDAR + AP Location ID Map the Innergy facility → Sage location / entity (e.g. 20-PA).
ARINVOICEITEM_PROJECTIDAR Project ID Get Sage project IDs onto the Innergy projects / work orders they correspond to.
ACCT_NO (revenue)AR GL revenue account Decide the 5,200-series account (furniture vs millwork), then set AR_REVENUE_ACCT_NO. Must never be 32000 (AP).

Things to know before importing

  • ACCT_NO (AR)Revenue account is still blank. RKL’s manual example posts revenue to a 5,200-series account (e.g. 50200 Furniture Sales), but which one depends on the unresolved furniture-vs-millwork split. It must never be 32000 (AP). The export warns until it’s set.
  • AR / AP acctAR posts to 12100 (Accounts Receivable control) — set from RKL’s example. Bills still use 32000 for the AP line; confirm that value too.
  • VENDOR_ID / CUSTOMER_IDBlank until configured. Set the Sage vendor/customer ID on the record’s External Id field in Innergy; it then flows into the export automatically.
  • SALES TAXHandled. Taxable invoices export two lines — a pre-tax revenue line plus a sales-tax line to 33500 (per RKL's IN-1002 example); untaxed invoices stay one line. Written as a plain GL line (not the template's SUBTOTAL flag, which needs Account Labels) — verify against a Sage test import.
